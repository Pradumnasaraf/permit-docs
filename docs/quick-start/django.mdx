## Overview

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. It is built by experienced developers to help you build web applications quickly and efficiently. In this guide, we will quickly go through the steps to integrate Permit.io with Django. We will use the Permit's CLI to create a new environment and a policy with a template. Then we will use the Permit SDK to integrate it with our Django application. We will also setup the PDP (Policy Decision Point) server locally using Docker.

We will see how we can sync users and tenant, assigning roles easily with Permit Dashboard. Then we will see how we can enforce policies using the Permit SDK in our Django application. 

## Prerequisites

- Docker (for running the PDP server)
- Node.js (for running the Permit CLI)
- Permit.io account ([Follow this guide](/quickstart) to create an account)

## Getting started

To get started, we will  first create a new policy with a template using the Permit CLI. Assuming you have already created an account with Permit.io.

### Hands on with Permit CLI

To get started, we will first install the Permit CLI. To install the CLI, execute the below command in your terminal. We will be using CLI to create policies, running PDP server and other tasks.

```bash
npm install -g @permitio/cli
```

Once the installation is complete, execute `permit` to confirm the installation. Then to authenticate, execute `permit login`. This will open a browser window and ask you to login with your Permit account. After logging in, the CLI will be authenticated and ready to use. This will also login into default environment (a project can have multiple environments),we can change environment by executing `permit env select` and then select the environment we want to use.

Now, let's create a policy with a template. The benefit of using a template is that we can create a policy with pre-defined roles and resources with all the good practices. This will save us time and effort in creating a policy from scratch. To check the available templates, execute `permit template list`. Based on our need we can apply the template to a new policy by executing `permit template apply <template-name>`. 

![cli template list](/img/quick-start-guide/env-template.png)

We have used the **fga-tradeoffs** template to create a new policy.

Now, let's visit the Permit dashboard to check the created policy and also get the API key that we will use to authenticate our application when using the Permit SDK.

As we can see in the image below, CLI has created Resources, Roles and Policies. Now let's add **Read** permission to the **Viewer** to the **Visit** resource. It very easy to add permissions using the dashboard. We can also add more roles and resources as per our need.

![permit policy](/img/quick-start-guide/policy-editor-dash.png)

To get the API key, click on the **Projects** from the left sidebar, then click on the the three dots on on the environment we want to use and click on **Copy API Key** and store it in a safe place. This API key will be used to authenticate our application with Permit.io. If you are having any difficulty in getting the API key, follow this [guide](/overview/connecting-your-app/#1-get-your-permit-environment-api-key).

Once we are done let's move towards the implementation part.

### Django Integration

Once you have a Django application setup, we can integrate Permit.io with it. First, we need to install the Permit SDK. To install the SDK, execute the below command in your terminal.

```bash
pip install permit
```

Depending on your application name, under `settings.py` file, we have to add the installed app and url configurations. For this demo it's `permit_demo`. 

```python
# prmit_demo/settings.py
INSTALLED_APPS = [
    'django.contrib.contenttypes',
    'django.contrib.staticfiles',
]

# Allow all hosts for demo
ALLOWED_HOSTS = ['*']
ROOT_URLCONF = "permit_demo.urls"
```

Then under the `urls.py` file, we have to add the url configurations. 

```python
# permit_demo/urls.py
from django.urls import path
from . import views  # views.py is in the same folder

urlpatterns = [
    path("", views.root),
    path("sync-user", views.sync_user),
    path("create-tenant", views.create_tenant),
]
```

Then we need to create a new file called `views.py` under the same folder. This file will contain the views for our application. Below is a sample implementation of the Permit SDK in a application. We will understand each part of the code in detail below.

```python
import os
import json
from django.http import JsonResponse
from permit import Permit
from pydantic import BaseModel, EmailStr, ValidationError

permit = Permit(
    token=os.getenv("PERMIT_API_KEY"),
    pdp="<PDP_URL>",
)

class UserIn(BaseModel):
    email: EmailStr
    first_name: str
    last_name: str

class TenantIn(BaseModel):
    key: str
    name: str
    description: str | None = None

async def root(request):
    return JsonResponse({"message": "Hello, Permit-Django!"})

async def sync_user(request):
    try:
        data = json.loads(request.body)
        user = UserIn(**data)
    except ValidationError as e:
        return JsonResponse(e.errors(), status=422, safe=False)

    synced = await permit.api.users.sync({
        "key": user.email,
        "email": user.email,
        "first_name": user.first_name,
        "last_name": user.last_name,
    })
    return JsonResponse(synced.dict() if hasattr(synced, "dict") else vars(synced), status=201)

async def create_tenant(request):
    try:
        data = json.loads(request.body)
        tenant = TenantIn(**data)
    except ValidationError as e:
        return JsonResponse(e.errors(), status=422, safe=False)

    created = await permit.api.tenants.create({
        "key": tenant.key,
        "name": tenant.name,
        "description": tenant.description,
    })
    return JsonResponse(created.dict() if hasattr(created, "dict") else vars(created), status=201)
```

At the top of the file, we have imported the required libraries. We have imported `os` to access the environment variables, `json` to parse the request body, `JsonResponse` from Django to return JSON responses, `Permit` from the Permit SDK to interact with Permit.io, and `BaseModel`, `EmailStr`, and `ValidationError` from Pydantic to validate the request body. Then we have initialized the Permit SDK with the API key, which is stored in the `PERMIT_API_KEY` environment variable. And the `pdp` URL which is the URL of the PDP server. For now we can leave it empty, we will look and setup the PDP server later.


In this above code, we have defined three endpoints. The first endpoint is the root endpoint which returns a simple message. The second endpoint is `/sync-user` which will sync the user with Permit.io. We are using the `sync` method to sync the user with Permit.io. The `sync` method takes a dictionary with the user details and returns a response. We are using Pydantic to validate the request body and return errors if any.

The third endpoint is `/create-tenant` which will create a tenant in Permit.io. We are using the `create` method to create the tenant. The `create` method takes a dictionary with the tenant details and returns a response.

## Policy Check

Now, our application is ready by syncing the user and we have policy created. Let's enforce and check the policy. To enforce the policy we use Permit's PDP (Policy Decision Point) to check if the user has access to the resource. So first let's setup the PDP.

To start a PDP we can use the Permit's CLI. To start a PDP, execute the below command in your terminal. Make sure you have Docker installed in your machine as this will start a PDP container locally on your machine.

```bash
permit pdp run
```

One you execute the above command, it will start a PDP server locally on your machine. You can check the logs to see if the PDP server is running successfully. The PDP server will be running on port `7766` by default. You can also change the port. 

![permit pdp run](/img/quick-start-guide/create-pdp.png)


One the PDP Server is running, update the `pdp` URL in the Django application. The URL will be `http://localhost:7766` if you are running the PDP locally. And create another endpoint to check our enforcement. This is how the updated code will look like.

```python
.
.
class PermissionCheck(BaseModel):
    user: str
    action: str
    resource: str
.
.
permit = Permit(
    token=os.getenv("PERMIT_API_KEY"),
    pdp="http://localhost:7766",  # Update if your PDP URL differs
)
.
.
async def check_permission(request):
    try:
        data = json.loads(request.body)
        check = PermissionCheck(**data)
    except ValidationError as e:
        return JsonResponse(e.errors(), status=422, safe=False)

    permitted = await permit.check(check.user, check.action, check.resource)
    return JsonResponse(
        {"message": "User is permitted"} if permitted else {"message": "User is not permitted"},
        status=200 if permitted else 403
    )
```

And then we need to add the new endpoint to the `urls.py` file. 

```python
# permit_demo/urls.py
.
urlpatterns = [
    .
    .
    path("check-access", views.check_permission),  # New endpoint to check access
]
```

In this code, we have added a new endpoint `/check-access` which will check if the user has access to the resource. We have defined a new model `PermissionCheck` to validate the request body.

We are using the `check` method to check if the user has access to the resource. The `check` method takes three parameters: `user`, `action` and `resource`. The `user` is the unique key of the user, `action` is the action that the user is trying to perform and `resource` is the resource that the user is trying to access.
The `check` method returns a boolean value, if the user is permitted to access the resource, it returns `True`, otherwise it returns `False`.

Now, everything is ready, let's restart the Django server and check the access. To check the access.As we have created a user and gave him access to the resource via the dashboard, we can check the access using curl.

![curl check access](/img/quick-start-guide/check-perm-pass.png)

As we can see in the image above, the user is permitted to access the resource. Let's remove the access from the dashboard and check again. 

![curl check access](/img/quick-start-guide/check-perm-fail.png)

Now this time we can see that the user is not permitted to access the resource. This is how we can enforce policies using Permit.io with Django.
